# Auto Mode グラフ表示改善 実装計画書（改訂版）

## 1. 概要
Auto Mode実行時に表示されるエネルギー推移グラフ（折れ線グラフ）の視認性を向上させるため、X軸（横軸）のスケール挙動を変更します。
当初の提案（固定値50）を取り下げ、**「対象となる全原子数（置換/削除の総試行回数）」に基づいて動的にX軸の最大値を設定**するように実装します。
これにより、データが少なくてもグラフが左から右へ自然に伸びていく進捗可視化を実現します。

## 2. 目的
*   **正確な進捗表示:** 実行予定の総ステップ数をグラフの全幅として確保し、現在の進捗状況を視覚的に正確に表現します。
*   **動的なスケール調整:** 処理対象が少ない場合（例: 5原子）も多い場合（例: 100原子）も、常に適切なスパンでグラフが表示されるようにします。

## 3. 対象ファイル
1.  `static/js/auto_mode_chart.js`
2.  `templates/index.html`

## 4. 実装詳細

### 4.1. `static/js/auto_mode_chart.js`
`reset()` メソッドを拡張し、総データ点数（`totalPoints`）を受け取れるようにします。

*   **変更箇所:** `AutoModeChart.reset()`
*   **変更内容:**
    *   引数 `totalPoints`（デフォルト50）を追加。
    *   `this.maxDataPoints`（スライディングウィンドウサイズ）を `totalPoints` に更新。
    *   Chart.jsの `options.scales.x` に対して `min: 0` および `max: totalPoints - 1` を設定し、X軸の描画範囲を固定。

### 4.2. `templates/index.html`
Auto Modeの各処理（置換・削除）のループ開始前に行われるグラフのリセット呼び出し時に、対象原子数を引数として渡します。

*   **変更箇所:**
    *   `handleOptimalDeletion` 関数内の `window.autoModeChart.reset()` 呼び出し
    *   `handleOptimalSubstitution`（または該当する最適化ループ）関数内の `window.autoModeChart.reset()` 呼び出し
*   **変更内容:**
    *   `window.autoModeChart.reset(targetAtoms.length);` に変更。

## 5. 検証手順

1.  **少数データの確認:**
    *   処理対象が少ない元素（例: 5個程度）を選択してAuto Modeを実行。
    *   グラフの右端が「5個目のデータ」に対応する位置になり、1個完了時点では全体の1/5程度まで線が伸びていることを確認。
2.  **多数データの確認:**
    *   処理対象が多い元素を選択して実行。
    *   データ点数に応じてX軸が適切に設定され、スライディング（古いデータが消える挙動）が発生せず、全履歴が表示されることを確認。
