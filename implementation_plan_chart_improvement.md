# Auto Mode グラフ表示改善 実装計画書

## 1. 概要
Auto Mode実行時に表示されるエネルギー推移グラフ（折れ線グラフ）の視認性を向上させるため、X軸（横軸）のスケール挙動を変更します。
Y軸（縦軸）については、微細なエネルギー変化を捉えるために有効な現在の「オートスケール（自動伸縮）」仕様を維持します。

## 2. 目的
*   **進捗の直感的な把握:** データ点数が少ない初期段階において、グラフを全幅に引き伸ばすのではなく、左から右へデータが積み上がっていく（伸びていく）表現にすることで、探索プロセスの時間経過と進捗を直感的に理解できるようにします。
*   **視覚的な安定性:** データ追加ごとの横幅変動をなくし、安定したグラフ描画を実現します。

## 3. 対象ファイル
*   `static/js/auto_mode_chart.js`

## 4. 実装詳細

### 4.1. X軸（横軸）のスケール固定
コンストラクタで定義されている `this.maxDataPoints`（デフォルト50）を利用し、X軸の表示範囲を常に固定します。

*   **変更箇所:** `initChart()` メソッド内の `options.scales.x` 設定
*   **変更内容:**
    *   `min` を `0` に設定
    *   `max` を `this.maxDataPoints - 1`（例: 49）に設定
    *   これにより、データ点が1つや2つの場合でも、X軸は常に50点分の幅を確保して描画されます。

### 4.2. Y軸（縦軸）のスケール設定
*   **方針:** 現状維持（オートスケール）
*   **理由:** 探索終盤の微小なエネルギー変化（meVオーダー）をユーザーが確認できるようにするため、データの最大値・最小値に合わせて自動的に拡大表示される現在の仕様が最適であると判断しました。

## 5. 検証手順

1.  **初期表示の確認:**
    *   Auto Modeを開始し、データが1〜2点追加された時点で、グラフが左端に寄って表示され、右側に空白があることを確認する（全幅に引き伸ばされていないこと）。
2.  **進行中の挙動確認:**
    *   データが増えるにつれて、グラフの線が右側へ伸びていくことを確認する。
3.  **ウィンドウサイズ到達時の挙動確認:**
    *   データが50点を超えた際、古いデータが左へ消え、最新データが右端に追加される（スライディングウィンドウ）挙動が維持されていることを確認する。
